5. Create `frontend/src/hooks/use-sse.ts` with `'use client'` directive:
   - Export function `useSSE()` that manages an EventSource connection.
   - Inside a `useEffect` (with cleanup):
     a. Create `new EventSource('/api/stream/prices')`
     b. `es.onopen` -> call `setConnectionStatus('connected')`
     c. `es.onmessage` -> `JSON.parse(event.data)` as `Record<string, PriceData>`, call `updatePrices(data)`, call `setConnectionStatus('connected')`
     d. `es.onerror` -> check `es.readyState`: if `EventSource.CONNECTING` (0) call `setConnectionStatus('reconnecting')`, if `EventSource.CLOSED` (2) call `setConnectionStatus('disconnected')`
     e. Cleanup function calls `es.close()`
   - Get `updatePrices` and `setConnectionStatus` from `usePriceStore` via selectors (not full store subscription).
   - Dependency array: `[updatePrices, setConnectionStatus]` — these are stable references from Zustand.

| SSE stream -> client | Price data received from same-origin backend via EventSource |
| API response -> client | Portfolio data received from same-origin fetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | SSE onmessage handler | accept | Same-origin policy enforced by browser; data is non-sensitive simulated prices; React auto-escapes all rendered values |
| T-01-02 | Information Disclosure | Portfolio fetch | accept | Single-user app with no auth by design; no sensitive PII; simulated money only |
| T-01-03 | Tampering | JSON.parse of SSE data | mitigate | Parse inside onmessage handler; malformed JSON throws and is caught by EventSource error handling; no dangerouslySetInnerHTML used anywhere |
