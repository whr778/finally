## Connection Status Indicator

Three visual states for the `ConnectionDot` component:

| State | Dot Color | Text | Trigger |
|-------|-----------|------|---------|
| Connected | `#3fb950` (green) | LIVE | `EventSource.onopen` fires, or first `onmessage` received |
| Reconnecting | `#d29922` (yellow) | CONNECTING | `EventSource.onerror` fires and readyState is CONNECTING |
| Disconnected | `#f85149` (red) | OFFLINE | `EventSource.onerror` fires and readyState is CLOSED |

**Layout:** Dot (8px circle) + 4px gap + text label. Right-aligned in header. Dot uses CSS `border-radius: 50%` and `background-color`. Text uses Label style (12px, regular weight).

**E2E contract:** Element with `data-testid="connection-dot"` must contain the text "LIVE" when the SSE connection is active.

**Source:** CONTEXT.md D-06, D-07, E2E spec `startup.spec.ts` line 39-44, `sse-resilience.spec.ts` line 8-11

## SSE Data Contract

**Endpoint:** `GET /api/stream/prices`

**Event format:** Each SSE `data:` line contains a JSON object keyed by ticker:

```json
{
  "AAPL": {
    "ticker": "AAPL",
    "price": 190.50,
    "previous_price": 190.25,
    "timestamp": 1713600000.123,
    "change": 0.25,
    "change_percent": 0.1314,
    "direction": "up"
  },
  "GOOGL": { ... }
}
```

**Frontend handling:**
1. Create `EventSource("/api/stream/prices")`
2. On `message` event: parse `JSON.parse(event.data)`, update Zustand price store
3. On `open`: set connection status to "connected"
4. On `error`: set connection status based on `readyState` (CONNECTING = reconnecting, CLOSED = disconnected)
5. EventSource auto-reconnects (server sends `retry: 1000`)

**Source:** Backend `stream.py`, `models.py` PriceUpdate.to_dict()

**Note:** `watchlist-row-{TICKER}` is checked in startup.spec.ts:33-37 but belongs to Phase 2 implementation. Phase 1 may render watchlist rows as simple placeholder text or defer to Phase 2. If the E2E test suite runs against Phase 1 code, these tests will fail -- that is expected until Phase 2 is complete.
