## Component Contracts

### 1. Portfolio Heatmap (Treemap)

**Placement:** Replaces `<PlaceholderPanel title="Portfolio Map" phaseNote="Coming in Phase 4" />` in AppShell. Second panel in the `col-span-2 grid grid-cols-3 gap-4` bottom row.

**Library:** Recharts `<Treemap>` component with custom `content` prop.

**Data source:** `usePortfolioStore` positions array. Each position provides `ticker`, `quantity`, `avg_cost`, `current_price`, `unrealized_pnl`, `pct_change`.

**Data transformation:** Map positions to treemap data format:
```
{
  name: position.ticker,
  size: position.quantity * (position.current_price ?? position.avg_cost),  // portfolio weight by value
  pct_change: position.pct_change ?? 0,
  unrealized_pnl: position.unrealized_pnl ?? 0
}
```

**Recharts Treemap Configuration:**
- `<Treemap>` with `width` and `height` from `<ResponsiveContainer>`
- `dataKey="size"` -- tiles sized by position market value (portfolio weight)
- `content` prop: custom React component rendering each tile
- `isAnimationActive={false}` -- disable animation for 2Hz data refresh performance
- `aspectRatio={1}` -- square-ish tile aspect ratio

**Custom Tile Content (Claude's Discretion Resolution):**

The `content` prop receives `{ x, y, width, height, name, pct_change }` for each tile. Render:

```
+-------------------+
| AAPL              |  (ticker: 14px mono, white, top-left)
| +1.05%            |  (pct_change: 12px Inter, white, below ticker)
+-------------------+
```

- Background fill: color from P&L color scale table above
- Text: `#e6edf3` for both ticker and percentage
- Ticker: 14px JetBrains Mono, regular weight
- P&L %: 12px Inter, regular weight, format `+X.XX%` / `-X.XX%`
- Tile border: `1px solid rgba(0,0,0,0.3)` to separate adjacent tiles
- Minimum tile dimension: if `width < 40` or `height < 30`, show ticker only (hide P&L %)
- `data-testid="tile-{TICKER}"` on each tile's outer `<g>` or `<rect>` element
- `onClick` handler: call `setSelectedTicker(name)` -- same as watchlist row click behavior (D-16)
- `cursor: pointer` on hover

**Empty state (D-04):**
- Text: "No positions -- buy shares to see your portfolio map"
- Style: 12px Inter, `#7d8590`, centered in panel

**Source:** CONTEXT.md D-01 through D-06, D-16; E2E trading.spec.ts line 94 (`tile-AAPL`)

### 2. P&L Line Chart

**Placement:** Replaces `<PlaceholderPanel title="P&L" phaseNote="Coming in Phase 4" />` in AppShell. Third panel in the `col-span-2 grid grid-cols-3 gap-4` bottom row.

**Library:** Recharts `<AreaChart>` with gradient fill.

**Data source:** `GET /api/portfolio/history` returning `SnapshotOut[]` where each entry is `{ total_value: number, recorded_at: string }` (ISO timestamp string).

**Data fetching:** Fetch on mount, then poll every 30 seconds via `setInterval`. Store snapshots in component-local state (not Zustand -- this data is only used by this component).

**Recharts AreaChart Configuration:**

```jsx
<ResponsiveContainer width="100%" height="100%">
  <AreaChart data={snapshots} margin={{ top: 8, right: 8, bottom: 0, left: 0 }}>
    <defs>
      <linearGradient id="pnlGradient" x1="0" y1="0" x2="0" y2="1">
        <stop offset="5%" stopColor="#209dd7" stopOpacity={0.3} />
        <stop offset="95%" stopColor="#209dd7" stopOpacity={0} />
      </linearGradient>
    </defs>
    <XAxis
      dataKey="recorded_at"
      tick={{ fontSize: 10, fill: '#7d8590' }}
      tickFormatter={formatTime}
      axisLine={false}
      tickLine={false}
    />
    <YAxis
      tick={{ fontSize: 10, fill: '#7d8590' }}
      tickFormatter={formatDollar}
      axisLine={false}
      tickLine={false}
      width={50}
    />
    <Area
      type="monotone"
      dataKey="total_value"
      stroke="#209dd7"
      strokeWidth={1.5}
      fillOpacity={1}
      fill="url(#pnlGradient)"
    />
  </AreaChart>
</ResponsiveContainer>
```

**Time axis formatting (Claude's Discretion Resolution):**
- Format: `HH:MM` for intraday snapshots (e.g., "14:30")
- Use `new Date(recorded_at).toLocaleTimeString('en-US', { hour: '2-digit', minute: '2-digit', hour12: false })`
- Show 4-6 ticks maximum (let Recharts auto-calculate interval)

**Value axis formatting:**
- Format: `$X.Xk` for values over 1000 (e.g., "$10.0k"), `$XXX` for under 1000
- Hide axis label (self-evident from context)

**No CartesianGrid** -- clean minimal look matching the terminal aesthetic.

**No Tooltip** -- data-dense terminal view; value is visible from Y-axis.

**Empty state (D-10):**
- Text: "Portfolio value chart will appear as data accumulates"
- Style: 12px Inter, `#7d8590`, centered in panel

**Source:** CONTEXT.md D-07 through D-11, D-17

### 3. Main Ticker Chart

**Placement:** Replaces the current placeholder `div` in AppShell (row 1, column 2 of the grid). Currently shows `"{selectedTicker} -- price chart"` text.

**Library:** Lightweight Charts 5.x (`createChart` + `AreaSeries`).

**Data source:** SSE price history accumulated since page load. The main chart needs the same per-ticker price history that sparklines use, but with timestamps. The component subscribes to `usePriceStore` for the selected ticker and accumulates `{ time, value }` pairs in a ref.

**Lightweight Charts Configuration (Claude's Discretion Resolution):**

```js
const chart = createChart(containerRef, {
  width: containerWidth,
  height: containerHeight,
  layout: {
    background: { type: 'solid', color: '#161b22' },
    textColor: '#7d8590',
    fontFamily: "'Inter', system-ui, sans-serif",
    fontSize: 10,
  },
  grid: {
    vertLines: { color: 'rgba(125, 133, 144, 0.1)' },
    horzLines: { color: 'rgba(125, 133, 144, 0.1)' },
  },
  timeScale: {
    timeVisible: true,
    secondsVisible: false,
    borderColor: 'rgba(125, 133, 144, 0.2)',
  },
  rightPriceScale: {
    borderColor: 'rgba(125, 133, 144, 0.2)',
  },
  crosshair: {
    mode: 0, // CrosshairMode.Normal
    vertLine: {
      width: 1,
      color: 'rgba(125, 133, 144, 0.4)',
      style: 3, // LineStyle.Dashed
    },
    horzLine: {
      width: 1,
      color: 'rgba(125, 133, 144, 0.4)',
      style: 3,
    },
  },
})
```

**Area Series Configuration:**

```js
const areaSeries = chart.addSeries(AreaSeries, {
  lineColor: '#209dd7',
  lineWidth: 1.5,
  topColor: 'rgba(32, 157, 215, 0.3)',
  bottomColor: 'rgba(32, 157, 215, 0)',
  lastValueVisible: true,
  priceLineVisible: false,
  crosshairMarkerVisible: true,
  crosshairMarkerRadius: 3,
})
```

**Data format:** Each point is `{ time: unixTimestamp, value: price }`. Lightweight Charts expects `time` as a UTCTimestamp (seconds since epoch). Convert from the SSE `timestamp` field (already seconds).

**Chart header:** Display `"{selectedTicker} -- price chart"` above the chart canvas. This text must remain for E2E compatibility (`watchlist.spec.ts` line 16 checks for `"GOOGL -- price chart"`).

- Header text: 16px, Inter, semibold, `#e6edf3`
- `--` is an em dash character (`\u2014`)

**Resize behavior (Claude's Discretion Resolution):**
- Use a `ResizeObserver` on the chart container div to call `chart.resize(width, height)` when the container dimensions change
- Clean up the observer on component unmount

**When no ticker selected (D-14):**
- Text: "Click a ticker to view its chart"
- Style: 12px Inter, `#7d8590`, centered vertically and horizontally in the panel
- No Lightweight Charts instance created until a ticker is selected

**When ticker changes:**
- Clear the area series data (`areaSeries.setData([])`)
- Start accumulating new price points for the newly selected ticker
- Call `chart.timeScale().fitContent()` after setting initial data

**Source:** CONTEXT.md D-12 through D-15; E2E watchlist.spec.ts line 16

## Cross-Panel Interaction (D-16)

### Treemap Tile Click

Clicking a treemap tile calls the same `setSelectedTicker` callback used by watchlist rows:
1. Updates the main chart to display the clicked ticker's price data
2. Populates the trade bar ticker input with the clicked ticker

This uses the existing `selectedTicker` state in AppShell and the `onSelectTicker` prop pattern established in Phase 2.

**AppShell modification:** Pass `onSelectTicker={setSelectedTicker}` to the new `HeatmapPanel` component (same pattern as `WatchlistPanel`).

## Interaction States

### Treemap

| State | Visual |
|-------|--------|
| Empty (no positions) | Centered muted text: "No positions -- buy shares to see your portfolio map" |
| Populated | Colored tiles sized by value, green/red/gray by P&L |
| Tile hover | `cursor: pointer`, slight brightness increase (CSS `filter: brightness(1.15)`) |
| Tile click | Sets selectedTicker, updates main chart and trade bar |
| After trade | New/updated tiles appear on next `fetchPortfolio` cycle (5s poll or immediate after trade) |

### P&L Chart

| State | Visual |
|-------|--------|
| Empty (no snapshots) | Centered muted text: "Portfolio value chart will appear as data accumulates" |
| Populated | Blue area chart with gradient fill |
| New data point arrives | Chart re-renders with new data (every 30s poll) |

### Main Ticker Chart

| State | Visual |
|-------|--------|
| No ticker selected | Centered muted text: "Click a ticker to view its chart" |
| Ticker selected, accumulating | Area chart builds progressively as SSE prices arrive (~500ms intervals) |
| Ticker changed | Chart clears and begins accumulating for new ticker |
| Crosshair hover | Dashed crosshair lines with price/time readouts (Lightweight Charts built-in) |
