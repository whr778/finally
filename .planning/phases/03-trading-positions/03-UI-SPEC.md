- Ticker input: `readOnly`, populated by watchlist click
- Qty input: `disabled`
- Buy/Sell buttons: `disabled`

### Phase 3 Activation (D-05, D-06, D-07)

```
+-----------------------------------------------------------------------+
| [AAPL    ] [  2.5  ] [ Buy ] [ Sell ]                                |
|  BUY 2.5 AAPL @ $191.50                          (success, fades)    |
+-----------------------------------------------------------------------+
```

| Element | data-testid | Activation Change |

| Ticker input | `trade-ticker` | Remove `readOnly`. Allow `.fill()` from E2E and manual typing. Watchlist click still populates. |

| Input | Behavior |
|-------|----------|
| Ticker | Editable text. Uppercase on submit (backend uppercases anyway). Populated by watchlist click via `selectedTicker` prop. |
| Quantity | Number input. Accepts decimals for fractional shares. `step="any"` attribute. `min="0"`. No spinner styling (hide with CSS). |

### Button States

| Button | Default | Hover | During Trade |
|--------|---------|-------|-------------|
| Buy | `bg-success/50`, white text, rounded, `px-4 py-1.5` | `bg-success/70` brightness increase | No loading state (trades are instant) |

| Outcome | Ticker Field | Qty Field | Feedback |

## Trade Feedback Bar (D-01, D-02)

Inline status bar below the trade bar inputs, inside the trade bar container.

### Layout

```
+-----------------------------------------------------------------------+
| [AAPL    ] [  2.5  ] [ Buy ] [ Sell ]                                |
| BUY 2.5 AAPL @ $191.50                                               |
+-----------------------------------------------------------------------+
```

The feedback text appears on a second row inside the trade bar container, below the input row. Use flex-col on the outer container, with the input row as the first child and the feedback row as the second child.

### Success State

| Property | Value |
|----------|-------|
| data-testid | `trade-success` |
| Text color | `#3fb950` (success green) |
| Font | 12px, mono, regular (Label size, monospace for data feel) |
| Format | `{SIDE} {quantity} {TICKER} @ ${price}` |
| Example | `BUY 2.5 AAPL @ $191.50` |
| Auto-dismiss | Fade out after 3 seconds (use `setTimeout` to clear, CSS `transition: opacity 300ms`) |
| Content requirements | Must contain the ticker symbol AND the side (BUY or SELL) per E2E contract |

### Error State

| Property | Value |
|----------|-------|
| data-testid | `trade-error` |
| Text color | `#f85149` (danger red) |
| Font | 12px, mono, regular |
| Format | Display the backend `detail` field directly |
| Auto-dismiss | Fade out after 3 seconds |

### Feedback Data Source

The backend `POST /api/portfolio/trade` returns:
- **Success (200):** `{ ticker, side, quantity, price, executed_at }` -- use `side.toUpperCase()`, `quantity`, `ticker`, and `price` to format the success message
- **Error (400/404):** `{ detail: "error message" }` -- display `detail` as the error text

**Source:** CONTEXT.md D-01, D-02, E2E trading.spec.ts

| Column | Format | Examples |

| Avg Cost | 2 decimal places | `189.50` |
| Price | 2 decimal places | `191.50` |
| P&L | 2 decimal places with sign, dollar prefix | `+$20.00`, `-$5.30` |
| % Change | 2 decimal places with sign, percent suffix | `+1.05%`, `-3.12%` |

### P&L Color Logic

- Positive value (> 0): `#3fb950` (success green) for both P&L and % columns
- Negative value (< 0): `#f85149` (danger red) for both P&L and % columns
- Zero or null: `#e6edf3` (text primary)

### Empty State (Before Any Trades)

| Property | Value |
|----------|-------|
| Heading | "Positions" (same as panel title, already visible) |
| Body text | No positions yet. Execute a trade to get started. |
| Font | 12px, Inter, regular (Label style) |
| Color | `#7d8590` (text muted) |
| Alignment | Centered vertically and horizontally in the panel body area |

### Live Updates

Positions table reads from `usePortfolioStore` which already polls every 5 seconds. After a trade, `fetchPortfolio()` is called immediately (D-08), so positions appear without waiting for the next poll cycle.

### Sort Order

Alphabetical by ticker symbol (ascending). This is a discretion choice that provides a predictable, stable order.

### Behavior

Clicking a watchlist row populates the trade bar ticker input. This was already wired in Phase 2 via the `selectedTicker` prop. Phase 3 changes:

## Post-Trade State Refresh (D-08)

### Flow

1. User clicks Buy or Sell
2. Frontend validates locally (ticker non-empty, qty > 0)
3. `POST /api/portfolio/trade` with `{ ticker, quantity, side }`
4. On success: show success feedback, clear qty, call `fetchPortfolio()` immediately
5. On error: show error feedback, keep all fields
6. The existing 5-second `setInterval(fetchPortfolio, 5000)` in AppShell continues as a safety net

### Local Validation (Claude's Discretion)

| Condition | Error Message |
|-----------|---------------|
| Empty ticker | Enter a ticker symbol |
| Empty or zero quantity | Enter a quantity |
| Negative quantity | Enter a positive quantity |

These are shown in the `trade-error` element. Backend errors (insufficient cash, insufficient shares, no price available) are also shown in `trade-error`.

**Source:** CONTEXT.md D-08, E2E trading.spec.ts (empty qty test)
