## Price Flash Animation (D-01)

### Visual Effect

On each price update, the entire watchlist row briefly glows green (uptick) or red (downtick), then fades back to transparent over 500ms. Bloomberg terminal style -- background glow, not just text color change.

### CSS Implementation

```css
@keyframes flashUpBg {
  0% { background-color: rgba(63, 185, 80, 0.3); }
  100% { background-color: transparent; }
}
@keyframes flashDownBg {
  0% { background-color: rgba(248, 81, 73, 0.3); }
  100% { background-color: transparent; }
}

.flash-up, .flash-up-a, .flash-up-b {
  animation: flashUpBg 500ms ease-out forwards;
}
.flash-down, .flash-down-a, .flash-down-b {
  animation: flashDownBg 500ms ease-out forwards;
}
```

### Animation Replay Strategy

Browsers do not replay a CSS animation when the same class is reapplied. To handle consecutive same-direction updates:

1. Maintain a boolean ref (`flashToggle`) that alternates on each update
2. Apply BOTH the base class and the alternating suffix:
   - Uptick: `flash-up flash-up-a` then `flash-up flash-up-b`
   - Downtick: `flash-down flash-down-a` then `flash-down flash-down-b`
3. The base class (`flash-up` / `flash-down`) satisfies E2E selectors
4. The alternating suffix (`-a` / `-b`) forces the browser to restart the animation

**E2E constraint:** The E2E test `watchlist.spec.ts` checks for the CSS class `.flash-up` specifically. The implementation MUST apply a class named exactly `flash-up` on uptick.

**Source:** CONTEXT.md D-01, RESEARCH.md Pattern 2, E2E contract

## SVG Sparkline (D-02)

### Visual Specification

| Property | Value |
|----------|-------|
| Width | 60px |
| Height | 24px |
| Line color | `#209dd7` (accent blue) |
| Line width | 1.5px |
| Fill | Linear gradient, `rgba(32,157,215,0.3)` at top to `rgba(32,157,215,0)` at bottom |
| Render | SVG `<polyline>` for line, `<polygon>` for filled area |

### Data Accumulation

| Property | Value |
|----------|-------|
| Data source | `useTickerPrice(ticker)` -- each price update adds a point |
| Storage | `useRef<number[]>([])` per row (NOT in Zustand) |
| Max points | 60 |
| Downsampling | When exceeding 60 points, filter to every other point (preserves shape, halves count) |
| Minimum points for render | 2 (below 2, render empty `<svg>` element) |
| Re-render trigger | Increment a `useState` counter after pushing new point |

### SVG Gradient ID Uniqueness

Each sparkline instance must use a unique gradient `id` attribute. Use React 19 `useId()` hook to generate unique IDs per component instance.

**E2E constraint:** The test `sse-resilience.spec.ts` checks for an `svg` element inside `watchlist-row-{TICKER}`. The sparkline SVG must be rendered as a child of the watchlist row.

**Source:** CONTEXT.md D-02, RESEARCH.md Pattern 3

## Ticker Selection (WATCH-04)

### Behavior

Clicking a watchlist row selects that ticker for:
1. The main chart area (displays `"{TICKER} -- price chart"` text in the placeholder until Phase 4)
2. The trade bar ticker input (populates with the selected ticker symbol)

### State

Store the selected ticker as a string. Options: add `selectedTicker` to a Zustand store or lift state to AppShell. Keep minimal -- Phase 4 will build full chart wiring.

### E2E Constraints

- Clicking `watchlist-row-GOOGL` must cause text `"GOOGL -- price chart"` to appear on the page
- Clicking `watchlist-row-TSLA` must populate `trade-ticker` input with `"TSLA"`
